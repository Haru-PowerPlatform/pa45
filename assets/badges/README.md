# PA45 参加バッジ画像フォルダ

## フォルダ構成

```
assets/badges/
├── session-003/
│   └── badge.png     ← 第3回バッジ画像（ここに配置）
├── session-004/
│   └── badge.png     ← 第4回バッジ画像（ここに配置）
├── session-005/
│   └── badge.png     ← 第5回バッジ画像（ここに配置）
└── ...
```

## ルール

- **ファイル名は必ず `badge.png`（または `badge.jpg`）で統一**
- 各セッションのフォルダに1枚だけ配置する
- Power Automateフローは **バッジ画像が存在しない場合はメールを送信しない**（安全装置）
- 新しいセッションが追加されたら、フォルダを作成してバッジ画像を配置してからフローを有効化する

## Power Automateフローとの連携

OneDriveのパス（例）:
`/ドキュメント/GitHub/pa45/assets/badges/session-004/badge.png`

フローの動作順序:
1. Formsに回答が届く
2. **バッジ画像の存在確認**（ファイルが存在しない → 管理者にエラー通知して終了）
3. バッジ画像が確認できた場合のみ → 参加者にメール送信
