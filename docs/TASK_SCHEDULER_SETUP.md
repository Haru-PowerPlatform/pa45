# Windows タスクスケジューラ で release-flow.bat を定期実行

毎週木曜21:30（PA45開催後）に `release-flow.bat all` を自動実行する設定例。

---

## セットアップ手順（5分）

### 1. タスクスケジューラを開く
- `Win + R` → `taskschd.msc` → Enter
- または スタート → 「タスク スケジューラ」検索

### 2. 「基本タスクの作成」
- 右側のアクションペインから「**基本タスクの作成**」クリック

### 3. ウィザードで以下入力

| 項目 | 値 |
|---|---|
| 名前 | `PA45 Flow Auto Release` |
| 説明 | PA45 のフローを毎週自動エクスポートしてサイト反映する |

### 4. トリガー
- 「**毎週**」を選択
- 開始日時: 木曜 21:30
- 「**木曜日**」にチェック

### 5. 操作
- 「**プログラムの開始**」を選択

### 6. プログラム/スクリプト
```
C:\Users\isamu\Documents\pa45\scripts\release-flow.bat
```

### 引数の追加
```
all
```

### 開始（オプション）
```
C:\Users\isamu\Documents\pa45
```

### 7. 完了

---

## 確認・テスト

タスク作成後、左側の「タスク スケジューラ ライブラリ」から作ったタスクを右クリック → 「**実行**」で即時テスト。

ログは `C:\Temp\pa45\flow-zips\` の更新タイムスタンプで確認できる。

---

## 注意点

### PCがスリープ中だと動かない
- タスクのプロパティ → 「**条件**」タブ → 「**タスクを実行するためにスリープを解除する**」にチェック

### ログイン中じゃないと動かない（既定）
- タスクのプロパティ → 「**全般**」タブ → 「**ユーザーがログオンしているかどうかにかかわらず実行する**」を選択
- → パスワード入力を求められる

### pac 認証が切れた場合
- pac CLIの認証は通常30〜90日で切れる
- 切れた状態でタスク実行 → エクスポート失敗
- 月1回くらい手動で `pac auth list` 確認、切れてたら `pac auth create --environment "PA45 Demo"` で再認証

### git push が失敗する場合
- リモートに変更があると non-fast-forward で失敗
- タスク内で `git pull --rebase` を先にする方が安全（必要なら release-flow.bat に追記）

## 実行頻度の選択肢

| シナリオ | 推奨頻度 |
|---|---|
| 確実性最優先・運用負荷ゼロ | 毎週木曜 21:30（開催後） |
| 月初バックアップだけ | 月初の月曜 9:00 |
| 開催直前にもチェックしたい | 毎週水曜 18:00 ＋ 木曜 21:30 |

---

## 削除・無効化

不要になったらタスク スケジューラ ライブラリから右クリック → 「**削除**」または「**無効化**」。
