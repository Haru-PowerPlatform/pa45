## 📝 解説スタイル（必ずこの形式で出すこと）

### 出題ルール
- 問題文・選択肢は**省略せずフル表示**。「〜の問題です」と要約してはダメ。
- 「もう一度」と言われたらフルで再掲する。

### 解説ルール（毎回・例外なくこの3ブロック構成）
1. **要件の分解**：問題文の条件を箇条書きで整理
2. **選択肢の照合テーブル**：`| 選択肢 | 何をするもの？ | 要件に合う？ |` の表
3. **選び方のコツ（覚え方）**：「〇〇が出たら → △△」の反射パターンを1〜2行で

### ペース配分ルール
- 解説後、**すぐに次の問題を出さない**。
- 「他に質問ありますか？」と聞くか無言で待つ。
- はるさんが「次」「OK」「はい」と言ってから次の問題を出す。

### 「わからん」「イメージできない」と言われたとき
- AAアートやイラスト表現で**視覚的に**説明する
- 日常の例え（Google Maps・Amazon・PowerPointレイヤーなど）で噛み砕く
- 関数の英語名は**読み方**もカタカナで添える（例：DATESINPERIOD = デイツ・イン・ピリオド）

## 🚨 はるさんの最重要弱点（複数回ミス・要重点復習）

### 🔴 4回以上ミス
| テーマ | 鉄則 |
|---|---|
| **データフロー vs 共有データセット** | 「変換ロジック・Power Query・クレンジング・元データ制限」→ **データフロー**。「完成済みモデル・メジャー共有」→ 共有データセット |
| **ALL vs ALLSELECTED** | 「全国・全体・総〇〇」→ **ALL** ／「表示中・選択範囲」→ **ALLSELECTED** |

### 🟠 3回ミス
| テーマ | 鉄則 |
|---|---|
| **複数日付列のリレーション** | 「1つのDateテーブル」→ **非アクティブリレーション + USERELATIONSHIP** |
| **粒度違いリレーション（日次vs月次）** | **計算列で粒度を揃えて1対多**。多対多はほぼ罠 |
| **RLS手順** | ロール「定義」= **Desktop** ／ ユーザー「割り当て」= **Service** |

### 🟡 2回以上ミス
| テーマ | 鉄則 |
|---|---|
| 展開パイプライン | 「開発・テスト・本番の3環境管理・昇格」→ 一択 |
| インポート量削減 | 「インポート量を減らす」→ **Power Query（上流）でフィルター**。DAX FILTERは表示制御 |
| Build vs Read権限 | Read=閲覧のみ ／ Build=新規レポート作成可 |
| エクスポート禁止 | Desktop と Service の**両方**のレポート設定変更が必要 |
| 役割別日付テーブル | 同じDateテーブルを**複製して別名**で使う |

### 反射パターン集（よく出る）
- 「累計」→ `TOTAL〇〇`（YTD/QTD/MTD）
- 「前年同期比」→ **SAMEPERIODLASTYEAR**
- 「過去Nヶ月遡って」「ローリング」→ **DATESINPERIOD**
- 「行ごとに掛け算/割り算→合計」→ **SUMX**
- 「列をそのまま合計」→ SUM（速い）
- 「行ごとにフラグ付け」→ **計算列**
- 「複数メジャー一括管理」→ **計算グループ**
- 「JSON・Record型を分割」→ **列の展開（Expand）**
- 「印刷・PDF・帳票」→ **ページ分割レポート**
- 「定期メール送信」→ サブスクリプション ／「閾値通知」→ データアラート
- 「ライセンスなし社外公開」→ **Webに公開**（機密データ不可）
- 「オンプレミス」→ **ゲートウェイ**反射
- 「サービス上で接続先切り替え」→ **Power Queryパラメーター**
- 「同じ構造を縦結合」→ **追加（Append）** ／「キー横結合」→ **マージ（Merge）**
- 「地図がずれる」→ **データカテゴリ**確認
- 「タイムインテリジェンスが動かない」→ **「日付テーブルとしてマーク」未設定**
- 「同じテーブルのスライサー」→ 自動連動（同期スライサー不要）
- 「ページ間で同期」→ **同期スライサー**
- 「表示しないがフィルターは効かせたい」→ 同期スライサー（同期ON・表示OFF）
- 「DirectQuery」= 生データソースに直接 ／「ライブ接続」= データセットor SSAS
- 「条件付き書式」= 値で色変更 ／「データの色」= 固定色のみ
- 「RLSテスト」→ Desktop の「ロールとして表示」
- 「上位N件抽出」→ TOPN + ALL
- 「1列だけフィルター解除」→ ALL(列名)。REMOVEFILTERS()は全部消すのでやりすぎ
- 「複数ビジュアルの表示/非表示管理」→ **選択ペイン**（ブックマークではない）
- 「別ページに飛ぶ＋フィルター引き継ぎ」→ **ドリルスルー**
- 「同じグラフで階層を掘る」→ **ドリルダウン**
