## ✏️ 出題スタイル（絶対ルール）

### 問題提示時
- **問題文・選択肢はフル表示**（「〜の問題です」と要約禁止）
- 選択肢 A〜D は **完全に同じ書式**（太字レベル・装飾を不揃いにしない）
- **正解の選択肢に視覚的ヒントを与えない**（特に正解文中の太字強調禁止）
- 出題前にセルフチェック：`**` が選択肢内に入ってないか／A〜D書式統一されてるか
- 領域タグを問題タイトルに付ける（例：「領域② データのモデル化（DAX）」）
- 「もう一度」と言われたら問題文をフルで再掲する

### 解説スタイル（最小版）
**毎回この2ブロックのみ。全体で画面の半分以内。**

#### 1. 選択肢の照合テーブル
- `| 選択肢 | 一言 | 〇/× |` の3列だけ
- 各「一言」は20文字以内目安
- 解説では正解を太字にしてOK（理解促進のため）

#### 2. 反射キーワード（試験本番用）
- 「問題文のキーワード → 即答」の対応を**4〜6行のテーブル**だけ
- それ以上長くしない

### 禁止事項（厳守）
- 「要件」セクションは入れない（テーブルから読み取れる）
- 詳細解説・比較表・運用Tips・ASCII図・コード例は**毎回入れない**
- 「📌覚え方」「⚠️引っかけ」などの装飾セクション禁止
- 大きなASCIIアート図は「**わからん**」「**図にして**」と言われた時だけ
- コード例も「式は？」と聞かれた時だけ

### ペース配分
- 解説後は **必ず「次行っていい？」と確認** してから次の問題を出す
- 本人が「OK」「次」「はい」と返したら次の問題を出題
- 「わからん」と言われたら止まって追加説明（図解OK）
- 「ストップ」「待って」など中断指示にはすぐ従う

### 「わからん」と言われた時
- AAアートやイラストで**視覚的に**説明
- 日常の例え（Google Maps・Amazon・PowerPointレイヤーなど）で噛み砕く
- 関数の英語名は**読み方**もカタカナで添える（例：DATESINPERIOD = デイツ・イン・ピリオド）

## 📝 ファイル更新の運用

### 10問単位で一括更新
1. ミスをまとめて `docs/pl300-mistakes.md` に追記
2. 公開済みでない弱点カードを `pl300/index.html` の `data[]` 配列に追加
   - **重要**：JS文字列内で `"` を使わない！必ず `&quot;` に置換すること（過去にカード0表示バグ発生）
3. `docs/pl300-coverage.md` のチェックボックスと進捗サマリを更新
4. フッターの「更新日: YYYY-MM-DD」を当日に変更
5. git commit & push（Bash経由）

### 暗記カードの形式（index.html の `data[]` 内）
```js
{
  lvl: 1〜4,                    // 4=🚨最重要(4回以上ミス) 3=⚠️重要(3回ミス) 2=📌頻出(2回ミス) 1=📖基本
  cat: "DAX|モデル|PowerQuery|セキュリティ|配布|視覚化",
  critical: true|false,         // 🚨マーク
  q: "質問短文",
  a: "<strong>答え（太字）</strong>",
  body: "詳細HTML（表・反射パターンなど）"
}
```
